# Лабораторная работа 3: Динамические массивы в языке C

## Цель работы
Изучить работу с динамической памятью: функции malloc, calloc, realloc, free, создание и использование динамических массивов.

---

## Ответы на вопросы

### 1. Что делает malloc?

**malloc** (memory allocation) — выделяет заданное количество байт памяти в куче (heap).

```c
int *arr = (int*)malloc(n * sizeof(int));
```

Особенности:
- Выделяет память без инициализации (содержит мусор)
- При ошибке возвращает NULL
- Требует явного освобождения через free

### 2. Чем отличается malloc от calloc?

| malloc | calloc |
|--------|--------|
| Не инициализирует память | Заполняет память нулями |
| Принимает 1 аргумент (размер в байтах) | Принимает 2 аргумента (количество, размер элемента) |
| Работает быстрее | Работает медленнее |

```c
// malloc
int *arr1 = (int*)malloc(n * sizeof(int));

// calloc
int *arr2 = (int*)calloc(n, sizeof(int));  // элементы = 0
```

### 3. Что делает realloc?

**realloc** — изменяет размер ранее выделенного блока памяти.

```c
// Увеличение массива
int *tmp = realloc(arr, new_size * sizeof(int));
if (tmp != NULL) {
    arr = tmp;
}
```

Особенности:
- Может изменить адрес блока памяти
- Сохраняет старые данные
- Новые элементы не инициализируются
- При ошибке возвращает NULL

**Важно:** Нельзя сразу присваивать результат в тот же указатель — можно потерять данные при ошибке.

### 4. Что произойдет при утечке памяти?

**Утечка памяти** — когда выделенная память не освобождается.

```c
void leak() {
    int *arr = malloc(100 * sizeof(int));
    // free(arr) не вызван — утечка!
}
```

Последствия:
- Память остается занятой до завершения программы
- При многократных утечках — исчерпание памяти
- Программа может зависнуть или упасть

### 5. Зачем нужен free?

**free** освобождает ранее выделенную динамическую память.

```c
free(arr);
arr = NULL;  // Рекомендуется обнулить указатель
```

Правила:
- Каждый malloc/calloc/realloc должен иметь парный free
- После free указатель становится висячим (dangling pointer)
- Обнуление указателя после free предотвращает ошибки

### 6. Что произойдет при двойном освобождении памяти?

**Двойное освобождение** — вызов free для одного и того же указателя дважды.

```c
free(arr);
free(arr);  // ОШИБКА! Неопределенное поведение
```

Последствия:
- Неопределенное поведение (undefined behavior)
- Возможен сбой программы (segmentation fault)
- Повреждение структур данных менеджера памяти

### 7. Как проверить, что malloc вернул NULL?

```c
int *arr = malloc(n * sizeof(int));

if (arr == NULL) {
    printf("Ошибка: не удалось выделить память\n");
    // Обработка ошибки
    return 1;
}
```

Всегда проверяйте результат malloc/calloc/realloc!

### 8. В чем отличие статического и динамического массива?

| Статический массив | Динамический массив |
|-------------------|---------------------|
| Размер известен при компиляции | Размер определяется во время выполнения |
| Создается в стеке | Создается в куче (heap) |
| Автоматически освобождается | Требует ручного free |
| Нельзя изменить размер | Можно изменить realloc |
| int arr[10]; | int *arr = malloc(10 * sizeof(int)); |

## Ключевые правила работы с динамической памятью

1. **Всегда проверяйте NULL** после malloc/calloc/realloc
2. **Каждый malloc → free** — не забывайте освобождать память
3. **Обнуляйте указатель** после free: `arr = NULL;`
4. **Используйте временный указатель** с realloc
5. **Не используйте память после free** — висячий указатель
6. **Не освобождайте дважды** — двойное освобождение
7. **Используйте sizeof** — не хардкодьте размеры типов
