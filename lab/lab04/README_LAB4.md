## 📚 Ответы на вопросы

### 1. Что такое файл в языке C?

**Файл** — это последовательность байтов, с которой программа работает через поток. Поток представляется указателем типа `FILE*`.

### 2. Что такое FILE*?

**FILE*** — это указатель на структуру, содержащую информацию о файле:
- Дескриптор файла
- Буфер обмена
- Текущая позиция
- Режим открытия
- Флаги ошибок и EOF

### 3. Какие режимы открытия файла существуют?

| Режим | Описание | Файл должен существовать? |
|-------|----------|---------------------------|
| `"r"` | Чтение текста | ✅ Да |
| `"w"` | Запись текста (создает/перезаписывает) | ❌ Нет |
| `"a"` | Дозапись в конец текста | ❌ Нет |
| `"rb"` | Чтение бинарного | ✅ Да |
| `"wb"` | Запись бинарного (создает/перезаписывает) | ❌ Нет |
| `"ab"` | Дозапись бинарного | ❌ Нет |
| `"r+"` | Чтение и запись | ✅ Да |
| `"w+"` | Чтение и запись (создает/перезаписывает) | ❌ Нет |
| `"a+"` | Чтение и дозапись | ❌ Нет |

### 4. Чем отличается текстовый файл от бинарного?

**Текстовый файл:**
- ✅ Содержит символы в виде текста (ASCII/UTF-8)
- ✅ Можно просмотреть в текстовом редакторе
- ✅ Переводы строк преобразуются (`\n` ↔ `\r\n` на Windows)
- ✅ Режимы: `"r"`, `"w"`, `"a"`

**Бинарный файл:**
- ✅ Содержит данные в их внутреннем представлении
- ✅ Записываются байты как есть
- ✅ Используется для структур и массивов
- ✅ Режимы: `"rb"`, `"wb"`, `"ab"`

### 5. Что делают функции putc и getc?

```c
// Запись одного символа в файл
int putc(int c, FILE *stream);

// Чтение одного символа из файла
int getc(FILE *stream);  // Возвращает EOF при конце файла
```

**Пример:**
```c
FILE *fp = fopen("test.txt", "w");
putc('A', fp);  // Записывает 'A'
fclose(fp);

fp = fopen("test.txt", "r");
char ch = getc(fp);  // Читает 'A'
```

### 6. Что делают fgets и fputs?

```c
// Чтение строки (максимум n-1 символов)
char *fgets(char *str, int n, FILE *stream);

// Запись строки в файл
int fputs(const char *str, FILE *stream);
```

**Отличие от gets/puts:**
- `fgets` безопасна (проверяет размер буфера)
- `fgets` сохраняет `\n` в конце строки
- `fputs` не добавляет `\n` автоматически

### 7. Что делают fprintf и fscanf?

```c
// Форматированная запись в файл
int fprintf(FILE *stream, const char *format, ...);

// Форматированное чтение из файла
int fscanf(FILE *stream, const char *format, ...);
```

**Пример:**
```c
// Запись
fprintf(fp, "Имя: %s, Возраст: %d, Рост: %.2f\n", name, age, height);

// Чтение
fscanf(fp, "%s %d %f", name, &age, &height);
```

### 8. Что делают fread и fwrite?

```c
// Чтение блока данных
size_t fread(void *ptr, size_t size, size_t count, FILE *stream);

// Запись блока данных
size_t fwrite(const void *ptr, size_t size, size_t count, FILE *stream);
```

**Параметры:**
- `ptr` — указатель на буфер
- `size` — размер одного элемента в байтах
- `count` — количество элементов
- `stream` — файловый поток

**Пример для структуры:**
```c
struct Person p;
fwrite(&p, sizeof(struct Person), 1, fp);
fread(&p, sizeof(struct Person), 1, fp);
```

### 9. Что делают fseek и ftell?

```c
// Перемещение указателя в файле
int fseek(FILE *stream, long offset, int whence);

// Получение текущей позиции
long ftell(FILE *stream);
```

**Параметр `whence`:**
- `SEEK_SET` — от начала файла
- `SEEK_CUR` — от текущей позиции
- `SEEK_END` — от конца файла

**Пример определения размера файла:**
```c
fseek(fp, 0, SEEK_END);  // Переход в конец
long size = ftell(fp);   // Получаем размер в байтах
fseek(fp, 0, SEEK_SET);  // Возврат в начало
```

### 10. Что делают feof и ferror?

```c
// Проверка конца файла
int feof(FILE *stream);   // Неноль если достигнут EOF

// Проверка ошибок
int ferror(FILE *stream);  // Неноль если была ошибка
```

**Пример использования:**
```c
while (!feof(fp)) {
    ch = getc(fp);
    if (ferror(fp)) {
        printf("Ошибка чтения!\n");
        break;
    }
}
```

### 11. Почему важно закрывать файл?

1. **💾 Сброс буфера** — данные из буфера записываются на диск
2. **🔓 Освобождение ресурсов** — закрывается дескриптор файла
3. **🤝 Доступ другим программам** — файл становится доступен
4. **⚡ Экономия памяти** — освобождается структура FILE
5. **🛡️ Предотвращение потери данных** — гарантия записи

## 📝 Пример использования

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    FILE *fp;

    // Открытие файла
    fp = fopen("data.txt", "w");
    if (fp == NULL) {
        printf("Ошибка открытия файла!\n");
        return 1;
    }

    // Запись данных
    fprintf(fp, "Hello, World!\n");

    // Закрытие файла
    fclose(fp);

    return 0;
}
```

## ⚠️ Частые ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `Segmentation fault` | Работа с NULL-указателем | Проверять результат fopen |
| Данные не записались | Нет fclose | Всегда закрывать файл |
| Мусор в файле | Бинарный режим без "b" | Использовать "wb", "rb" |
| Потеря данных | Перезапись "w" вместо "a" | Использовать "a" для дозаписи |
