# Personal Data Manager (Менеджер персональных данных)

## Цель проекта

Консольное приложение на языке C для управления персональными данными с использованием:
[Почему были использованы такие технологии](readmehowwork.md)  
  Объяснение выбора технологий и подходов.
* **Структур данных** и динамической памяти (`malloc/realloc/free`)
* **Указателей на функции** для организации меню (без `switch/if`)
* **Стандартных библиотек**: `stdio.h`, `stdlib.h`, `string.h`, `time.h`, `ctype.h`
* **Файлового ввода/вывода**: бинарные (`fwrite/fread`) и текстовые файлы
* **Алгоритмов**: `qsort` для сортировки, линейный поиск [Как работает сортировка](READMEhowworksort.md)

## Как работает проект
"[Person.h](../Person.c) Мы используем заголовочный файл для разделения интерфейса и реализации. Это позволяет менять код в .c файлах, не трогая другие модули. Особенно важен extern для глобальных переменных — иначе каждый файл создаст свою копию переменной и произойдёт конфликт компоновщика."

"[main.c](../main.c) Вместо традиционного switch-case мы используем массив указателей на функции. Это позволяет добавлять новые пункты меню, просто добавляя элемент в массив, без изменения логики выбора. Это прямое применение указателей на функции — мощной возможности C, которая лежит в основе полиморфизма в ООП."

"[person.c](../person.c) Мы реализовали динамический массив с автоматическим расширением. При заполнении capacity удваивается через realloc. Это стратегия 'amortized constant time' — дорогая операция копирования происходит редко, поэтому в среднем вставка работает за O(1). Также показываем использование time.h для вычисления реального возраста относительно системного времени."

"[input.c](../input.c) Для валидации имени используем функцию isalpha из библиотеки ctype.h, которая проверяет принадлежность к категории букв с учётом локали. Также реализована очистка буфера ввода для предотвращения зацикливания при вводе нечисловых данных в числовые поля."

"[file_io.c](../file_io.c) Реализовано два формата хранения: текстовый (для человеческого чтения) и бинарный (для эффективности). В бинарном режиме используем fwrite/fread, которые выполняют прямое копирование байт из памяти на диск без преобразований. Это быстрее и компактнее, но требует соблюдения порядка записи/чтения и переносимости между платформами (endianness)."

"[search.c](../search.c) Реализован линейный поиск с временной сложностью O(n). Для сравнения строк используем strcasecmp, которая игнорирует регистр символов, делая поиск удобнее для пользователя. Для больших данных можно было бы использовать бинарный поиск (требует предварительной сортировки), но для учебного проекта линейный достаточен."

"[sort.c](../sort.c) Для сортировки используем стандартную функцию qsort из stdlib.h, которая реализует алгоритм быстрой сортировки (quick sort) со средней сложностью O(n log n). Мы передаём в неё компараторы — функции сравнения, которые сообщают qsort, как упорядочивать элементы. Это пример использования функций обратного вызова (callback functions)."

"[stats.c](../stats.c) Демонстрируем использование макроса _Generic из стандарта C11 для создания универсальной функции вывода, которая автоматически выбирает спецификатор формата (%d, %.2f, %s) в зависимости от типа аргумента на этапе компиляции. Также показываем работу с time.h: получаем текущий timestamp, конвертируем в локальное время и вычисляем возраст как разницу текущего года и года рождения."

"[utils.c](../utils.c) Вынесли вспомогательные функции (оформление таблиц, пауза экрана) в отдельный модуль для соблюдения принципа разделения ответственности (Single Responsibility Principle)."
### Архитектура меню

Главное меню реализовано через массив указателей на функции:

```c
void (*menu_functions[])(void) = {
    add_person,
    print_all,
    sort_by_name,
    ...
};
```

Пользователь вводит номер пункта, и программа вызывает функцию по индексу массива **без использования `switch` или `if`**.

## Управление памятью

Данные хранятся в динамическом массиве структур `Person`, который автоматически расширяется (обычно в 2 раза) при заполнении.

---

## Основные функции

* **Добавление** — ввод с клавиатуры с валидацией
  (имя — только буквы, числа > 0)

* **Вывод** — форматированная таблица всех записей

* **Сортировка** — `qsort`:

  * по имени
  * по возрасту

* **Поиск**:

  * по имени (точное совпадение)
  * по диапазону возраста

* **Статистика**:

  * средний возраст
  * максимальный рост
  * реальный возраст (через `time.h`)

* **Файлы**:

  * сохранение и загрузка
  * бинарный и текстовый формат

## Как открыть и скомпилировать

### Требования

* Компилятор C (`gcc`, `clang`, `MSVC`)
* Поддержка стандарта **C11**

---

### Компиляция (терминал)

#### Linux / Mac

```bash
gcc -std=c11 -o manager main.c person.c input.c file_io.c search.c sort.c stats.c utils.c
./manager
```

#### Windows (MinGW / Cygwin)

```bash
gcc -std=c11 -o manager.exe main.c person.c input.c file_io.c search.c sort.c stats.c utils.c
manager.exe
```

---

### IDE (Code::Blocks, CLion, VS Code)

1. Создай новый проект C
2. Добавь все `.c` файлы и `person.h`
3. Установи стандарт **C11**
4. Собери и запусти проект
