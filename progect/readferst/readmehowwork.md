# Архитектурные решения проекта (для защиты)

## Общая концепция

Проект построен по принципу **модульного программирования** с разделением. Каждый модуль отвечает только за свою задачу, что делает код:
- Легче для понимания (не нужно читать 500 строк кода сразу)
- Легче для отладки (ошибка сразу локализуется в модуле)
- Легче для расширения (можно добавить новый модуль, не трогая старые)

---

## Почему такое разделение на файлы?

### 1. `main.c` — Точка входа + Диспетчеризация
**Почему отдельно:** Главный файл должен содержать только логику запуска и управления потоком выполнения. Он не знает "как" выполнять операции, знает только "что" нужно выполнить.

**Ключевая фишка:** Использование массива указателей на функции вместо switch-case.
- **Преимущество switch:** При добавлении нового пункта меню не нужно менять логику выбора, просто добавляем элемент в массив.
- **Сложность O(1):** Независимо от количества пунктов, выбор происходит за константное время (одно обращение по индексу).
- **Полиморфизм:** Это имитация виртуальных методов из ООП в процедурном C.

### 2. `person.c` — Работа с данными (Модель)
**Почему отдельно:** Это "модель" данных (в терминах MVC). Здесь описано, как создавать, хранить и отображать персональные данные.

**Ключевая фишка:** Динамический массив с автоматическим расширением.
- **Почему не статический:** Неизвестно количество записей заранее. Статический массив[100] — это либо ограничение функционала, либо трата памяти.
- **Стратегия удвоения:** При заполнении capacity удваивается. Это дает амортизированную сложность O(1) на вставку (дорогая операция realloc происходит редко).

### 3. `input.c` — Валидация (Отделение бизнес-логики от ввода)
**Почему отдельно:** Ввод данных и их обработка — разные задачи. Валидация может меняться (например, добавить проверку на максимальную длину имени), не затрагивая логику работы со структурами.

**Ключевая фишка:** Функции `clear_input_buffer()` и `validate_name()` с использованием `ctype.h`.
- **Защита от переполнения:** `fgets` вместо `scanf("%s")` предотвращает переполнение буфера.
- **Проверка символов:** `isalpha()` работает с учетом локали (может поддерживать Unicode в будущем).

### 4. `file_io.c` — Инфраструктурный слой
**Почему отдельно:** Работа с файловой системой — это "внешний мир". Если завтра решим хранить данные в базе SQLite вместо файлов — меняем только этот файл.

**Ключевая фишка:** Двойной формат (текстовый + бинарный).
- **Текстовый:** Человекочитаемый, можно открыть в блокноте, переносим между системами.
- **Бинарный:** Максимальная скорость и компактность (нет преобразования чисел в строки), но не читаем человеком.

**Автоматическое управление директориями:** Папка `data/` создается автоматически, чтобы отделить пользовательские данные от кода программы (аналог "Документы" или "AppData" в профессиональных приложениях).

### 5. `search.c` — Алгоритмы поиска
**Почему отдельно:** Поиск — это отдельная бизнес-операция. Можно добавить индексацию или бинарный поиск (для отсортированного массива), не меняя другие модули.

**Ключевая фишка:** `strcasecmp()` для поиска без учета регистра.
- **UX:** Пользователь может ввести "alice", "Alice" или "ALICE" — результат будет одинаковым.

### 6. `sort.c` — Алгоритмы сортировки
**Почему отдельно:** Сортировка — чистая функция (не зависит от контекста). Можно заменить qsort на mergesort или heapsort, не трогая другие файлы.

**Ключевая фишка:** Использование стандартной `qsort` вместо собственной реализации.
- **Надежность:** Код стандартной библиотеки оттестирован десятилетиями.
- **Оптимизация:** Разработчики компилятора встроили оптимизации (introsort — гибрид quicksort + heapsort) для защиты от худшего случая O(n²).

### 7. `stats.c` — Аналитика
**Почему отдельно:** Статистика — это отчетность, она не меняет данные (read-only). Отдельный модуль позволяет легко добавлять новые метрики (минимальный рост, медиана возраста и т.д.).

**Ключевая фишка:** Макрос `_Generic` (C11).
- **Типобезопасность:** Компилятор выбирает правильный printf на этапе компиляции, а не во время выполнения.
- **Универсальность:** Одна "функция" `print_value` работает с int, float, string без перегрузки (которой нет в C).

### 8. `utils.c` — Вспомогательные функции
**Почему отдельно:** Это "служебный" код, который мог бы использоваться в любом другом модуле. Выносим общие утилиты, чтобы не дублировать код.
